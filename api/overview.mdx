---

The Gemma 3 inference engine exposes a local HTTP API that mirrors the OpenAI chat completions interface. Because the API is wire-compatible with OpenAI's specification, you can point any OpenAI client library at `http://localhost:8000/v1` and interact with the local Gemma model without changing your application code.

## Available endpoints

The server exposes three routes:

| Method | Path | Purpose |
|--------|------|---------|
| `POST` | `/v1/chat/completions` | Generate a chat completion (streaming or non-streaming) |
| `GET` | `/healthz` | Liveness probe — confirms the server process is running |
| `GET` | `/readyz` | Readiness probe — confirms the model has finished loading |

## Authentication

No authentication is required. The server is designed for local use and does not validate API keys. When using the OpenAI Python SDK, pass any non-empty string for `api_key`.

## Starting the server

```bash
python -m openai_api.run
```

The server binds to `http://localhost:8000` by default. Model weights are downloaded or loaded from disk during startup; the `/readyz` endpoint returns `200` once the model is ready to serve requests.

## Example: OpenAI Python SDK

```python
from openai import OpenAI

client = OpenAI(base_url="http://localhost:8000/v1", api_key="unused")

response = client.chat.completions.create(
    model="gemma-3-270m-it",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

<Note>
  The only accepted value for `model` is `"gemma-3-270m-it"`. Passing any other model name returns a validation error.
</Note>
