Direct inference lets you run the Gemma 3 model without starting any server. Running `main.py` loads the model and tokenizer on your local device, applies sampling, and streams generated tokens to stdout — useful for experimenting with prompts or verifying model behavior before deploying the API.

## How direct inference works

`main.py` wires together five responsibilities: device detection, weight download, tokenizer setup, prompt preparation, and streaming generation. There is no HTTP layer — the script calls `generate_text_stream` directly and prints each decoded token as it is produced. The model runs entirely on the selected device using `torch.inference_mode()` to avoid gradient tracking overhead.

## RunConfig fields

All runtime parameters live in the `RunConfig` dataclass at the top of `main.py`. Edit these values to change model behavior before running the script.

```python
@dataclass
class RunConfig:
    choose_model: str = "270m"
    use_instruct_model: bool = True
    prompt: str = "Give me a short introduction to large language models."
    max_new_tokens: int = 180
    temperature: float = 0.8
    top_p: float = 0.9
    top_k: int = 50
    repetition_penalty: float = 1.1
```

| Field | Default | Description |
|---|---|---|
| `choose_model` | `"270m"` | Model size selector. Currently only `"270m"` is supported. |
| `use_instruct_model` | `True` | When `True`, downloads `gemma-3-270m-it` and applies the chat template. |
| `prompt` | `"Give me a short introduction to large language models."` | The user message passed to the model. |
| `max_new_tokens` | `180` | Maximum number of tokens the model can generate before stopping. |
| `temperature` | `0.8` | Sampling temperature. Lower values produce more deterministic output. |
| `top_p` | `0.9` | Nucleus sampling threshold. Tokens outside the top cumulative probability are excluded. |
| `top_k` | `50` | Restricts sampling to the top-k highest-probability tokens at each step. |
| `repetition_penalty` | `1.1` | Penalizes previously generated tokens to reduce repetition. Values above `1.0` apply the penalty. |

## Run direct inference

```bash
python main.py
```

The script downloads model weights and the tokenizer on first run (requires approved Hugging Face access to `google/gemma-3-270m` / `google/gemma-3-270m-it`). Subsequent runs reuse local files from the `gemma-3-270m-it/` directory.

## Customize RunConfig

Open `main.py` and modify the `RunConfig` instantiation in `main()`. For example, to run the base model with a custom prompt and a higher token budget:

```python
def main() -> None:
    run = RunConfig(
        use_instruct_model=False,
        prompt="The history of transformer architecture begins with",
        max_new_tokens=300,
        temperature=0.6,
        repetition_penalty=1.05,
    )
    ...
```

You can also adjust only sampling parameters and keep everything else at its default:

```python
run = RunConfig(
    prompt="Explain paged KV cache in one paragraph.",
    temperature=0.2,
    top_k=20,
)
```

## How generate_text_stream works

`generate_text_stream` is a Python generator that yields one token tensor per iteration. Each call runs a single forward pass through the model, extracts the last-position logits, applies the repetition penalty, then samples the next token using temperature scaling, top-k filtering, and nucleus (top-p) sampling in that order. Generation stops when the EOS token is produced or `max_new_tokens` steps are exhausted.

```python
for token in generate_text_stream(
    model=model,
    token_ids=token_ids,
    max_new_tokens=run.max_new_tokens,
    eos_token_id=tokenizer.eos_token_id,
    temperature=run.temperature,
    top_p=run.top_p,
    top_k=run.top_k,
    repetition_penalty=run.repetition_penalty,
):
    token_id = token.squeeze(0).tolist()
    print(tokenizer.decode(token_id), end="", flush=True)
```

The generator uses `past_kv` caching between steps so only the most recently generated token is fed back as input on each decode step.

## Device selection

`get_device()` checks for available hardware in priority order: CUDA first, then Apple MPS, then CPU as a fallback.

```python
def get_device() -> torch.device:
    if torch.cuda.is_available():
        return torch.device("cuda")
    if torch.backends.mps.is_available():
        return torch.device("mps")
    return torch.device("cpu")
```

No configuration is required — the device is selected automatically at startup.

## GPU memory reporting

When running on CUDA, `main.py` resets peak memory statistics before generation and prints the high-water mark after the last token:

```python
if torch.cuda.is_available():
    torch.cuda.reset_peak_memory_stats()

# ... generation loop ...

if torch.cuda.is_available():
    print(f"\n\nGPU memory used: {calc_gpu_gb(torch.cuda.max_memory_allocated())}")
```

The value is formatted in gigabytes, for example `GPU memory used: 0.54 GB`. This measurement covers model activations and the KV cache accumulated during the generation loop.

<Note>
When `use_instruct_model=True`, `main.py` wraps your prompt with the Gemma chat template before tokenizing:

```python
prompt = apply_chat_template(run.prompt) if run.use_instruct_model else run.prompt
```

`apply_chat_template` formats the string as:

```
<start_of_turn>user
{user_text}<end_of_turn>
<start_of_turn>model
```

This tells the instruct-tuned model to respond in assistant mode. When `use_instruct_model=False`, the raw prompt string is passed directly to the tokenizer, which is appropriate for base model completions rather than chat-style interaction.
</Note>
